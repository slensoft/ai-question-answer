# 快速选项配置说明

## 概述

每个思考框架的问题现在支持配置快速选择选项，让用户可以快速填充答案，提升使用体验。

## 数据结构

### Question 类型定义

```typescript
export interface Question {
  text: string;              // 问题文本（必填）
  quickOptions?: string[];   // 快速选择选项（可选）
  placeholder?: string;      // 输入框提示文字（可选）
}
```

### Methodology 类型

```typescript
export interface Methodology {
  // ... 其他字段
  questions: (string | Question)[]; // 兼容旧格式和新格式
}
```

## 快速选项设计原则

### 1. 数量建议
- 建议 6 个选项（2行 x 3列的网格布局）
- 最少 3 个，最多 9 个

### 2. 选项内容
- 简短明确（2-6 个字）
- 覆盖常见场景
- 互相独立，不重复
- 符合问题类型

### 3. 问题类型与选项匹配

#### 原因类问题（为什么）

## 用户交互

### 选择行为
- 点击快速选项后，内容会追加到答案框
- 多次点击会用分号分隔：`选项1; 选项2; 选项3`
- 用户可以继续手动编辑答案

### 显示逻辑
- 只有配置了 `quickOptions` 的问题才显示快速选择区域
- 如果 `quickOptions` 为空数组或未配置，不显示该区域
- 保持界面简洁，避免不必要的元素

## 配置新方法论

### 步骤

1. 打开 `src/lib/methodology-data.ts`
2. 找到要配置的方法论
3. 将 `questions` 数组中的字符串改为对象格式
4. 添加 `quickOptions` 和 `placeholder`

### 示例

**修改前：**
```typescript
questions: [
  'What - 这是什么问题？',
  'Why - 为什么会这样？'
]
```

**修改后：**
```typescript
questions: [
  {
    text: 'What - 这是什么问题？',
    quickOptions: ['技术问题', '流程问题', '沟通问题', '资源问题', '质量问题', '时间问题'],
    placeholder: '描述具体的问题情况...'
  },
  {
    text: 'Why - 为什么会这样？',
    quickOptions: ['资源不足', '沟通不畅', '流程缺陷', '技术限制', '时间紧迫', '目标不清'],
    placeholder: '分析问题的根本原因...'
  }
]
```

## 注意事项

1. **向后兼容**：系统同时支持字符串和对象格式，旧数据无需立即迁移
2. **可选配置**：`quickOptions` 和 `placeholder` 都是可选的
3. **灵活性**：不同方法论可以有不同的选项数量和内容
4. **用户体验**：快速选项是辅助工具，不应限制用户的自由输入
