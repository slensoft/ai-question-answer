# 方法论平台页面结构

## 页面架构

### 1. 首页 `/methodology`
- **功能**: 平台入口，提供导航和使用指南
- **内容**:
  - 平台介绍
  - 三个主要入口卡片（场景选择、决策树、所有方法论）
  - 使用建议

### 2. 按场景选择 `/methodology/scenarios`
- **功能**: 根据实际场景（工作、学习、创新、决策）快速找到合适的方法论
- **交互流程**:
  1. 用户选择场景类型（工作/学习/创新/决策）
  2. 弹出该场景下的具体需求列表
  3. 点击需求后跳转到推荐的方法论详情页

### 3. 智能决策树 `/methodology/decision-tree`
- **功能**: 通过回答问题引导用户找到最合适的方法论
- **交互流程**:
  1. 回答"你需要什么？"
  2. 根据回答继续深入提问
  3. 最终推荐一个方法论并跳转到详情页

### 4. 所有方法论 `/methodology/all`
- **功能**: 方法论资源库，可浏览和搜索所有15种方法论
- **特性**:
  - 搜索功能
  - 分类筛选（思维模型、结构化提问、深度追问等）
  - 卡片式展示

### 5. 方法论详情 `/methodology/[slug]`
- **功能**: 具体方法论的学习和实践页面
- **内容**:
  - 方法论介绍
  - 实践表单（问题描述、思考框架、反思）
  - 学习记录历史
- **示例 URL**:
  - `/methodology/5W2H`
  - `/methodology/SCQA`
  - `/methodology/STAR`

## 导航栏

所有页面顶部都有统一的导航栏，包含：
- 🏠 首页
- 🎯 按场景选择
- 🌳 智能决策树
- 📚 所有方法论

当前页面的导航项会高亮显示。

## 设计理念

1. **引导式学习**: 不直接展示所有方法论，而是通过场景和决策树引导用户找到最合适的方法
2. **清晰的信息架构**: 每个页面职责单一，用户路径清晰
3. **灵活的访问方式**: 提供多种方式（场景、决策树、直接浏览）满足不同用户需求
4. **独立的 URL**: 每个方法论都有独立的 URL，支持分享和书签

## 用户流程

### 场景驱动流程
首页 → 按场景选择 → 选择场景 → 选择具体需求 → 方法论详情 → 开始实践

### 决策树流程
首页 → 智能决策树 → 回答问题 → 方法论详情 → 开始实践

### 直接浏览流程
首页 → 所有方法论 → 搜索/筛选 → 方法论详情 → 开始实践
