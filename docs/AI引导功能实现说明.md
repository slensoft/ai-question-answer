### 2. 融入的理论知识

#### 2.1 隐性知识 vs 显性知识
- 解释了为什么 AI 提问特别有效
- 提供了具体示例

#### 2.2 邓宁-克鲁格效应（核心理论）
- 完整的四阶段曲线可视化（使用真实图片）
- 每个阶段的详细说明：
  - ① 愚昧山峰：不知道自己不知道
  - ② 绝望之谷：知道自己不知道
  - ③ 开悟之坡：知道自己知道
  - ④ 持续平稳高原：知道边界
- 图片位置：`public/dunning-kruger-effect.png`

#### 2.3 信息熵减
- 可视化展示每个问题如何降低不确定性
- 从 30 个方法论 → 15 个 → 8 个 → 3 个

#### 2.4 苏格拉底式教学
- 对比传统方式 vs 引导式提问
- 说明为什么不直接给答案

## 🎨 设计亮点

### 1. 渐进式信息披露
- 默认不显示理论知识（避免吓跑用户）
- 点击"了解更多"才展开详细内容
- 符合"愚昧山峰"用户的心理

### 2. 视觉化理论
- DK 曲线用 SVG 绘制
- 信息熵用进度条展示
- 对比框用颜色区分好坏

### 3. 教育性选项
- 每个选项都有说明（括号内）
- 用户在选择时就在学习
- 例如："分析问题（找出原因、理解现状）"

### 4. 实时反馈
- 显示当前可能的方法论
- 显示置信度百分比
- 显示进度（第几题）

## 🚀 如何测试

### 1. 启动开发服务器
```bash
npm run dev
```

### 2. 访问页面
- 首页：http://localhost:3000
- AI 引导：http://localhost:3000/ai-guide

### 3. 测试流程
1. 点击首页的"🤖 AI 智能引导"按钮
2. 查看欢迎页的理论知识（点击"了解更多"）
3. 点击"开始提问"
4. 回答模拟问题
5. 查看实时反馈（右侧边栏）

## 📊 技术栈

- **框架**：Next.js 16 + React 19
- **语言**：TypeScript
- **样式**：CSS Modules + 自定义 CSS
- **动画**：CSS Animations
- **图表**：SVG（手写）

## 🎯 设计原则

### 1. 零门槛
- 不要求任何背景知识
- 文案简单易懂
- 视觉友好

### 2. 渐进式
- 从简单到复杂
- 从浅到深
- 可选择性学习

### 3. 教育性
- 在使用中学习
- 理论与实践结合
- 即时反馈

### 4. 趣味性
- 像玩游戏
- 有进度感
- 有成就感
