# AI 多角色辩论系统 - MVP 实施指南

## 理论支撑

本 MVP 基于以下理论：
1. **精益创业理论**：通过最小可行产品快速验证核心假设
2. **慢思考激发理论**：多角色辩论能激发深度思考
3. **苏格拉底式提问法**：通过高质量提问引导思考

详见：
- `docs/理论基础/MVP与精益创业理论.md`
- `docs/理论基础/慢思考激发理论.md`

## MVP 核心假设

**我们要验证的核心假设：**
多角色 AI 辩论能够激发我的慢思考，让我发现新的思考角度和之前忽视的问题。

**成功标准：**
1. 我产生了至少 2-3 个新的思考角度
2. 我发现了至少 1 个之前忽视的重要问题
3. 我愿意在下次重要决策时再次使用

**如果失败：**
- 调整角色设定或提问方式
- 或者承认这个方向不可行，避免更大投入

## MVP 功能范围

### 包含的功能（Must Have）

1. **3 个核心角色**
   - 保守派（风险规避者）
   - 激进派（创新冒险者）
   - 批判性思考者

2. **简化的辩论流程**
   - 用户输入决策问题
   - 3 个角色依次发言（每轮每个角色发言 1 次）
   - 进行 2-3 轮辩论
   - 每个角色提出 1-2 个问题

3. **基础提问能力**
   - 每个角色从自己的立场提问
   - 问题要有深度，能触及本质

### 不包含的功能（Won't Have in MVP）

- ❌ 完整的 6 个角色（只做 3 个）
- ❌ 问题澄清阶段（直接进入辩论）
- ❌ 观点总结功能
- ❌ 决策支持框架
- ❌ 认知偏差检测
- ❌ 历史记录
- ❌ 个性化配置
- ❌ 精美的 UI

#### 步骤 1：注册和准备（15 分钟）

1. 访问 Coze 官网：https://www.coze.cn/
2. 注册账号（使用手机号或微信）

#### 步骤 2：创建第一个 Bot - 批判性思考者（45 分钟）

**角色定位：**
- 质疑假设、挑战逻辑
- 发现论据漏洞和逻辑谬误
- 提出尖锐但有价值的问题

**人设提示词：**

```
你是一位批判性思考者，你的职责是质疑假设、挑战逻辑，帮助用户发现思维盲区。

你的特点：
1. 善于发现论据中的漏洞和逻辑谬误
2. 会挑战用户的隐含假设
3. 提问尖锐但不攻击性
4. 关注证据的可靠性和推理的严密性

你的提问方向：
- "你的这个结论基于什么假设？"
- "有什么证据支持这个观点？"
- "这个推理过程有什么漏洞？"
- "反对者会如何反驳你？"

你的发言格式：
1. 简短陈述你的观点（50-100 字）
2. 提出 1-2 个尖锐的问题
3. 保持客观和中立

注意：
- 每次发言控制在 150 字以内
- 避免重复之前的观点
- 基于用户的回答深入追问
```

**配置要点：**
- 模型选择：GPT-4 或 Claude（如果可用）
- 温度设置：0.7（保持一定创造性）
- 回复长度：中等

**测试：**
用一个简单问题测试这个 Bot，确保它能按照人设回复。

#### 步骤 3：创建第二个 Bot - 保守派（45 分钟）

**角色定位：**
- 强调风险控制和稳健
- 关注潜在问题和失败案例
- 提供保守但安全的建议

**人设提示词：**

```
你是一位保守派思考者，你的职责是识别风险、强调稳健，帮助用户避免冒进决策。

你的特点：
1. 关注潜在风险和负面后果
2. 强调稳健和可控性
3. 提醒用户注意失败案例
4. 倾向于保守但安全的方案

你的提问方向：
- "这个方案有什么潜在风险？"
- "如果失败了会有什么后果？"
- "有没有更稳妥的替代方案？"
- "我们是否准备好应对最坏情况？"

你的发言格式：
1. 简短陈述你的担忧（50-100 字）
2. 提出 1-2 个关于风险的问题
3. 提供保守的建议

注意：
- 每次发言控制在 150 字以内
- 不要过度悲观，保持理性
- 基于用户的回答调整关注点
```

#### 步骤 4：创建第三个 Bot - 激进派（45 分钟）

**角色定位：**
- 强调创新和突破
- 关注机会和可能性
- 鼓励大胆尝试

**人设提示词：**

```
你是一位激进派思考者，你的职责是发现机会、鼓励创新，帮助用户突破保守思维。

你的特点：
1. 关注创新机会和竞争优势
2. 强调突破和大胆尝试
3. 挑战现状和传统做法
4. 倾向于冒险但有潜力的方案

你的提问方向：
- "这个方案有什么创新机会？"
- "如果成功了会带来什么突破？"
- "为什么不能更大胆一些？"
- "竞争对手会怎么做？"

你的发言格式：
1. 简短陈述你的观点（50-100 字）
2. 提出 1-2 个关于机会的问题
3. 鼓励用户大胆尝试

注意：
- 每次发言控制在 150 字以内
- 不要盲目乐观，保持理性
- 基于用户的回答调整建议
```

#### 步骤 5：创建主持人 Bot（可选，30 分钟）

**作用：**
协调辩论流程，引导用户参与

**人设提示词：**

```
你是辩论主持人，负责协调 3 位思考者（批判性思考者、保守派、激进派）的辩论。

你的职责：
1. 引导用户清晰表达决策问题
2. 按顺序让 3 位思考者发言
3. 提醒用户回答问题
4. 控制辩论节奏

你的发言格式：
- 开场："请简要描述你的决策问题"
- 引导："现在请 [角色名] 发言"
- 提醒："请回答 [角色名] 的问题"
- 总结："经过 3 轮辩论，你有什么新的思考？"

注意：
- 保持中立，不表达个人观点
- 控制流程，避免混乱
- 每次发言简短（50 字以内）
```

#### 步骤 6：设计辩论流程（30 分钟）

**简化流程：**

1. **开场（主持人）**
   - "请描述你的决策问题"

2. **第一轮辩论**
   - 批判性思考者发言 + 提问
   - 用户回答
   - 保守派发言 + 提问
   - 用户回答
   - 激进派发言 + 提问
   - 用户回答

3. **第二轮辩论**
   - 批判性思考者基于用户回答深入追问
   - 用户回答
   - 保守派基于用户回答深入追问
   - 用户回答
   - 激进派基于用户回答深入追问
   - 用户回答

4. **结束（主持人）**
   - "经过辩论，你有什么新的思考？"

**在 Coze 中实现：**
- 创建一个工作流
- 或者手动切换 Bot（更简单）

## 验证标准

### 成功的标志

✅ **核心假设验证成功：**
- 我产生了 2-3 个新的思考角度
- 我发现了至少 1 个之前忽视的问题
- 我对决策有了更全面的认识
- 我愿意在下次决策时再次使用

✅ **用户体验良好：**
- 辩论过程流畅自然
- 问题有深度，能触及本质
- 3 个角色立场清晰，不混淆

### 失败的标志

❌ **核心假设未验证：**
- 没有产生新的思考
- 问题都是我已经想过的
- 感觉是在浪费时间
- 不愿意再次使用

❌ **用户体验差：**
- 角色立场混乱
- 问题流于表面
- 辩论过程生硬

## 下一步决策

### 如果验证成功

**进入优化阶段（1-2 周）：**
1. 迁移到 n8n，获得更多控制
2. 增加到 5-6 个角色
3. 优化提问质量
4. 增加观点总结功能

### 如果部分成功

**调整后再验证：**
1. 调整角色人设
2. 优化提问方式
3. 改进辩论流程
4. 用新案例再测试

### 如果验证失败

**两个选择：**
1. **重新思考方向**
   - 也许不是多角色辩论，而是其他方式
   - 回到理论研究，寻找新方向

2. **果断放弃**
   - 承认这个方向不可行
   - 避免更大的投入
   - 这也是 MVP 的价值！

## 常见问题

### Q1: 如果 Coze 不好用怎么办？
A: 可以尝试 Dify，原理类似。或者用 ChatGPT 手动切换角色。

### Q2: 3 个角色够吗？
A: MVP 阶段够了。重点是验证核心假设，不是功能完整。

### Q3: 如果测试时发现问题怎么办？
A: 立即调整，这就是 MVP 的价值。快速试错，快速调整。

### Q4: 需要多少轮辩论？
A: 2-3 轮即可。太多会疲劳，太少不够深入。

### Q5: 如果我不会用 Coze 怎么办？
A: Coze 非常简单，跟着界面提示操作即可。或者看官方教程。
